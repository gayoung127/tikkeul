## 문서 작성 원칙

- **문서를 쓸 수 없으면 이해하지 못한 것이다** — 코드 전에 문서를 먼저 쓴다
- **결정의 결과(what)와 과정(why)을 분리한다** — 설계 문서에는 현재 구조를, ADR에는 결정 근거를 담는다
- **구체적인 숫자 예시를 포함한다** — 추상적 설명보다 "BTC 0.01개를 5천만원에 매수" 같은 시나리오가 이해를 돕는다
- **문서 간 중복을 피하고 링크로 연결한다**

# 업비트 서비스 분석

> 이 문서는 국내 1위 암호화폐 거래소인 업비트를 분석한 문서입니다.
> 우리 서비스의 정책과 규칙은 [비즈니스 규칙](business-rules.md)에서 정의합니다.

---

## 1. 서비스 개요

### 1.1 업비트란?

- 두나무 주식회사가 2017년 10월 출시한 국내 1위 암호화폐 거래소
- 원화(KRW), 비트코인(BTC), 테더(USDT) 마켓 지원
- 케이뱅크와 실명계좌 연동 시스템
- 24시간 365일 운영

### 1.2 핵심 비즈니스 가치

- 안전한 디지털 자산 거래 환경 제공
- 실시간 시세 정보 제공
- 자금세탁 방지(AML) 및 고객확인(KYC) 준수

## 2. 사용자 플로우

### 2.1 회원가입 및 인증

```
[앱 설치]
    │
    ▼
[휴대폰 본인인증] → 국내 통신사 본인명의 필수
    │
    ▼
[은행/증권사 계좌 인증] → 1원 송금 후 입금자명 숫자 3자리 확인
    │
    ▼
[PIN 비밀번호 설정] → 6자리 + 생체인증(선택)
    │
    ▼
[닉네임 설정 + 이용약관 동의]
    │
    ▼
[고객확인절차(KYC)]
    ├─→ 유의사항 확인 및 약관 동의
    ├─→ 기본/필수 정보 입력 (영문명, 실거주지 등)
    ├─→ 신분증 인증 (주민등록증/운전면허증 촬영)
    └─→ 케이뱅크 실명계좌 연동 → 원화 거래 필수
            │
            ▼
        [KYC 완료 대기] → 담당부서 확인 (일정 시간 소요)
            │
            ▼
        [거래 가능 상태]
```

**주요 특징:**

- 만 19세 이상만 가입 가능
- 케이뱅크 계좌 연동 시에만 원화 거래 가능
- 2채널 인증(카카오톡/네이버/하나 인증서) 필수

### 2.2 원화 입금

```
[입금 요청]
    │
    ▼
[케이뱅크 계좌 확인]
    │
    ▼
[입금 한도 확인]
    │   - 1회: 1억원 (한도계좌: 500만원)
    │   - 1일: 5억원 (한도계좌: 500만원)
    │
    ▼
[은행 이체 실행] → 케이뱅크 → 업비트
    │
    ▼
[잔액 반영] → 즉시
```

### 2.3 원화 출금

```
[출금 요청]
    │
    ▼
[출금 가능 금액 확인]
    │   출금가능금액 = 총보유자산 - 거래대기금액 - 출금수수료
    │   거래대기금액 = 미체결 주문 + 출금대기금액
    │
    ▼
[출금 한도 확인]
    │   - 1회: 1억원 (한도계좌: 5천만원)
    │   - 1일: 5억원 (한도계좌: 2억원)
    │
    ▼
[2채널 인증] → [출금 처리] → [케이뱅크 계좌 입금]

⚠️ 은행 점검 시간(23:56 ~ 00:06) 출금 불가
```

### 2.4 디지털 자산 입금

```
[입금 주소 확인] → 코인별 고유 주소
    │
    ▼
[외부 지갑/거래소에서 전송]
    │
    ▼
[블록체인 컨펌 대기]
    │
    ▼
[입금 금액에 따른 처리]
    ├─ < 100만원  → 즉시 입금 반영
    └─ ≥ 100만원 (트래블룰 적용)
        ├─ VV 연동 거래소 → 자동 반영
        └─ 기타 → 추가 확인 필요 (계정주 확인, 입금 출처 소명)
```

### 2.5 디지털 자산 출금

```
[출금 가능 금액 확인]
    │
    ▼
[24시간 출금 지연제 확인] ⭐
    │   원화 입금 후 24시간 이내:
    │   해당 금액 상당의 디지털 자산 출금 제한
    │
    ▼
[출금 주소 입력]
    │
    ▼
[출금 금액에 따른 처리]
    ├─ < 100만원  → 출금주소 신청으로 확인된 주소만 가능
    └─ ≥ 100만원 (트래블룰 적용)
        ├─ VV 연동 거래소만 출금 가능
        ├─ 등록된 개인지갑만 출금 가능
        └─ 받는사람 정보 입력 필수
    │
    ▼
[출금 네트워크 확인] → [2채널 인증] → [블록체인 전송]
```

### 2.6 거래 (주문 ~ 체결)

```
[코인 선택] → [호가창 확인]
    │
    ▼
[주문 방식 선택]
    ├─ 지정가: 가격 + 수량 입력
    ├─ 시장가: 매수는 총액만, 매도는 수량만
    ├─ 예약: 감시가격 설정 → 도달 시 자동 주문
    └─ 최유리지정가: 접수 시점 최적 호가로 자동 지정 (IOC/FOK)
    │
    ▼
[주문 검증]
    │   - 주문총액 ≥ 최소주문금액
    │   - 잔액 충분 여부
    │   - 불리한 체결 방지 검증
    │
    ▼
[오더북 등록] → [매매 체결 엔진]
    │               │   1. 가격 우선 (매수: 높은가격, 매도: 낮은가격)
    │               │   2. 시간 우선 (먼저 제출한 주문 우선)
    │               ▼
    │           [체결 결과: 전량 / 부분 / 미체결]
    │
    ▼
[잔액 업데이트]
    ├─ 매수: 원화 감소, 코인 증가
    └─ 매도: 코인 감소, 원화 증가
```

### 2.7 주문 취소

```
[미체결 주문 조회]
    │
    ▼
[취소 요청]
    ├─ 취소 전 체결됨 → 취소 실패
    └─ 취소 성공 → 거래대기금액 해제 → 잔액 복구
```

시스템에 의한 자동 취소도 존재한다:
- 서비스 점검 시: 마지막 체결가보다 높은 매수주문 취소
- 코인 이벤트(하드포크, 에어드랍 등) 발생 시

## 4. 주문 유형

### 4.1 지정가 주문 (Limit Order)

- 가격과 수량을 직접 지정
- 지정 가격에 도달해도 시간 우선순위에 의해 미체결 가능
- 매수: 지정가보다 높은 가격에 체결되지 않음
- 매도: 지정가보다 낮은 가격에 체결되지 않음

### 4.2 시장가 주문 (Market Order)

- 가격 지정 없이 즉시 체결 (테이커 주문)
- 매수: 주문총액만 설정, 가장 낮은 매도호가부터 체결
- 매도: 주문수량만 설정, 가장 높은 매수호가부터 체결
- 자산 보호를 위해 거부될 수 있음

### 4.3 예약 주문 (Trigger Order)

- 감시가격 도달 시 자동 주문 실행 (지정가/시장가 모두 가능)
- 활용: 손절매, 돌파 매수

### 4.4 최유리지정가 주문

- 주문 접수 시점의 최적 호가로 자동 지정
- IOC(즉시 체결 후 잔여 취소) 또는 FOK(전량 체결 아니면 전량 취소) 조건 필수

### 4.5 TWAP 주문

- Time-Weighted Average Price
- 지정 기간 동안 주문 분할 접수, 시장 영향 최소화

## 5. 체결 규칙

### 5.1 가격-시간 우선 원칙 (Price-Time Priority)

1. **가격 우선**: 매수는 높은 가격, 매도는 낮은 가격 우선
2. **시간 우선**: 동일 가격이면 먼저 제출한 주문 우선

### 5.2 체결 예시

```
[오더북 상태]
매도 호가:
  100,000원 - 10 BTC
   99,000원 -  5 BTC  ← 최우선 매도호가 (best ask)

매수 호가:
   98,000원 -  8 BTC  ← 최우선 매수호가 (best bid)
   97,000원 - 12 BTC

[시장가 매수 주문: 10억원]
1차 체결: 99,000원 × 5 BTC = 495,000,000원
2차 체결: 100,000원 × 5.05 BTC = 505,000,000원
→ 총 10.05 BTC 매수 완료
```

## 7. 실시간 데이터 (WebSocket)

### 7.1 연결 정보

- Public: `wss://api.upbit.com/websocket/v1`
- Private: `wss://api.upbit.com/websocket/v1/private`
- Timeout: 120초 (ping/pong으로 유지)

### 7.2 구독 가능 데이터

| 타입 | 설명 | 엔드포인트 |
|------|------|-----------|
| ticker | 현재가 | public |
| orderbook | 호가 | public |
| trade | 체결 | public |
| candle | 캔들 | public |
| myOrder | 내 주문 | private |
| myAsset | 내 자산 | private |

### 7.3 구독 요청 형식

```json
[
  {"ticket": "UNIQUE_TICKET"},
  {"type": "ticker", "codes": ["KRW-BTC", "KRW-ETH"]},
  {"type": "orderbook", "codes": ["KRW-BTC"]},
  {"type": "trade", "codes": ["KRW-BTC"]}
]
```

## 8. 규제 및 보안

### 8.1 트래블룰 (Travel Rule)

- 100만원 이상 입출금 시 송수신인 정보 확인 필수
- VV(베리파이바스프) 솔루션 연동 거래소만 입출금 가능
- 미신고 해외 거래소와 거래 금지

### 8.2 24시간 출금 지연제

- 보이스피싱 등 금융사기 방지 목적
- 원화 입금 후 24시간 이내 해당 금액 상당 디지털 자산 출금 제한
- 거래 이력 기반 위험도 분석으로 예외 적용 가능

### 8.3 회원 등급별 한도

| 구분 | 일반 (1회/1일) | 한도계좌 (1회/1일) |
|------|----------------|-------------------|
| 원화 입금 | 1억 / 5억 | 500만 / 500만 |
| 원화 출금 | 1억 / 5억 | 5천만 / 2억 |
| 디지털 자산 출금 | 등급별 상이 | - |

### 8.4 보안 인증

- 2채널 인증: 카카오톡/네이버/하나 인증서
- PIN 비밀번호 6자리
- 생체 인증 (Face ID/Touch ID)
- 로그인 알림, 비정상 접근 차단

## 9. 도메인 용어 (Ubiquitous Language)

| 용어 | 영문 | 정의 |
|------|------|------|
| 호가 | Orderbook | 매수/매도 주문이 쌓여있는 가격대별 주문 현황 |
| 체결 | Trade / Execution | 매수와 매도 주문이 만나 거래가 성사되는 것 |
| 미체결 | Pending / Open | 아직 체결되지 않은 주문 |
| 거래대기금액 | Reserved Amount | 미체결 주문 + 출금 진행중 금액 |
| 주문가능금액 | Available Balance | 총 잔액 - 거래대기금액 |
| 메이커 | Maker | 오더북에 유동성을 제공하는 주문 (지정가) |
| 테이커 | Taker | 오더북의 유동성을 소비하는 주문 (시장가) |
| 1호가 | Best Bid/Ask | 최우선 매수/매도 호가 |
| 슬리피지 | Slippage | 예상 체결가와 실제 체결가의 차이 |
| KYC | Know Your Customer | 고객확인절차 (신원 확인) |
| AML | Anti-Money Laundering | 자금세탁방지 |
| 트래블룰 | Travel Rule | 가상자산 송수신 시 정보 전달 의무 |
